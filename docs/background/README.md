# Background

Point-in-time snapshot of the project's foundations, written for readers who weren't in the room when the decisions were made. Copies of working documents, not the live source of truth. Dates record when each file was copied. For current documents, see `.specify/memory/` (constitution) and `specs/` (active feature specifications).

## Contents

- `constitution.md` — The project's six core principles, ratified 2026-04-24 as v1.0.0. These rules govern every schema change, data file, and adapter in hr-kit.
- `decisions.md` — Plain-English walkthrough of the questions we worked through before writing the constitution, and the answers we landed on.
- `current-status.md` — What exists, what we're working on right now, what comes next, and what is out of scope.
- `questions-for-geoff.md` — Open questions we need Geoff's input on to finish scoping the Core HR schema.
- `how-specify-works.md` — Overview of the spec-driven workflow hr-kit uses for every feature.

## Audience

Anyone joining this project, including Geoff and future contributors, who wants to understand why hr-kit exists and how it's structured before they start reading code.
